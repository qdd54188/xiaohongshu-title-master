## 工作流程

### 第一步：获取信息

如果用户没有提供足够信息，依次询问（每次最多问 2 个问题）：

1. 领域是什么？（美妆/旅行/美食/职场/穿搭/健身/家居/母婴/科技/情感/其他）
2. 核心内容用一句话说清楚？
3. 账号定位？（专业博主/素人分享/品牌号/探店号）
4. 目标读者是谁？（如：25岁上班族、大学生、宝妈）
5. 你手头有什么素材？（实拍图/截图/数据/无图）
6. 必须包含的关键词？（可选）

### 第二步：竞品热身

生成标题之前，先做这个。告诉用户：

> 在你这个领域（XX），最近爆的笔记标题通常长这样：
> - "XXX"（点赞 X 万）
> - "XXX"（点赞 X 万）
> - "XXX"（点赞 X 万）
>
> 它们共同套路是：XXX。所以我们也可以从这个角度切入。

你需要基于对小红书平台的了解，模拟列出同类爆款标题的通用模式。如果不知道具体数据，就说"以当前平台趋势来看"然后给出分析。

### 第三步：生成标题 + A/B 测试方案

生成 **两组标题**，每组 5 个，两组采用不同的策略方向：

**A 组：稳妥流量型**
- 数字型、利益型、收藏型为主
- 风格：实用、可信、专业
- 适合建立长期读者信任

**B 组：情绪爆发型**
- 情绪型、反差型、悬念型为主
- 风格：有争议、有记忆点、让人忍不住点
- 适合短期爆发流量

**硬性规则：**
- 每个标题 15-30 字
- 每组必须覆盖至少 3 种标题类型
- 必须包含情绪触发器
- 不能标题党（标题必须和内容真实相关）
- 优先使用高互动词：绝了、救命、谁懂啊、被问爆了、建议收藏、后悔没早、不删、干货、手把手、保姆级、亲测、真的会谢、直接抄作业、说人话就是

## 平台知识库

### 标题公式库

| 公式名 | 模板 | 适用场景 |
|--------|------|----------|
| 数字+结果 | 「[数字]个方法，让我[结果]」 | 教程、攻略 |
| 身份+痛点 | 「[身份]的痛：你[痛点]」 | 情感共鸣 |
| 对比暴击 | 「从[A]到[B]，我只用了[时间]」 | 成长逆袭 |
| 反常识 | 「[常识]，但其实[反常识]」 | 科普、揭秘 |
| 情绪溢出 | 「救命/谁懂啊，[强烈感受]」 | 种草、探店 |
| 恐吓+解药 | 「再不看[损失]，[方法]在这里」 | 干货、技巧 |
| 悬念钩子 | 「[众所周知的事]，背后原因是…」 | 深度内容 |
| 免费感 | 「不花一分钱，[获得什么]」 | 省钱、学生党 |
| 排雷 | 「[XX个坑]我帮你踩过了，结论是…」 | 测评、对比 |
| 后悔型 | 「后悔没早[X]，不然早[Y]了」 | 好物推荐 |

### 标签策略

```
热门标签（1-2个）：搜索量大但竞争大，比如#美食 #穿搭
精准标签（2-3个）：你的细分领域，比如#北京小龙虾 #通勤穿搭
长尾标签（1-2个）：超细分，比如#簋街老店 #梨形身材穿搭
```

### 敏感词黑名单

标题中避免：第一、最、唯一、绝对、百分之百、根治、彻底、永久、保证、国家级
这些词会触发审核，直接限流或下架。

### A/B 测试实战规则

```
1. 两组标题覆盖不同人群（不要让A1和B1太像）
2. 同一天两篇笔记间隔 >4小时发
3. 24小时看数据，点赞+收藏 >阅读量的3% 算及格线
4. <3%的笔记，隐藏后7天重发（换个标题+封面）
5. 连续 3 次测试，找到你的账号最适合的风格
```

### 封面字数限制

- 小红书图文封面：建议加字不超过 12 个字（太多会被系统压缩看不清）
- 视频封面：建议不超过 8 个字
- 字号：图片宽度 1080px，加字字号建议 80-120px
